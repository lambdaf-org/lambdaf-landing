01
PROTOCOL · NO APP CODE
Claude CodePlain filesKnowledge graph

Overseer /overseer ↗

Tell it what you know about yourself, the people around you, and your work, then ask “what should I focus on this week?” and get an answer drawn from one graph of your goals, projects, and network.

Overseer is plain files plus one protocol (OVERSEER.md); Claude Code reads your corpus and folds it into a single graph of nodes and edges in brain/, with nothing to deploy and no schema to migrate. It never stores status; it records dated events and works out the current state when you ask. The index stays small and re-derivable, so a question costs about what it needs.

It pushes back: when what you say you want drifts from what your record shows, it says so.

Built by191-iota
02
SWISS LAW · NO APP CODE
Claude CodeSwiss federal lawRAGFedlex

Saul Swissman /saul-swissman ↗

Ask a question about Swiss federal law and get an answer where every statement carries the article, the SR number, the verbatim German wording, and the consolidation date.

Saul reads the statutes in corpus/ch/, routes to the governing articles through a small index in register/, and quotes them word for word; a statement without an article and a verbatim quote does not get made. There is no application code; SAUL.md defines how it works and Claude Code is the engine. When a provision is not in its corpus, it says so and points to Fedlex.

It quotes only the statute it opened this turn; when the corpus runs out, it names the gap and points to Fedlex.

Built by191-iota
03
RUST · LOCAL-ONLY
RustOllamafastembed

auspex /auspex-v1 ↗

Point it at your chat exports and get cited per-person profiles plus a real interaction graph, then re-run to see a ranked feed of what changed in your network.

A single Rust binary parses your message exports, runs them through a local Ollama model and fastembed retrieval, and serves an embedded D3 graph at localhost:8765. Every claim cites the exact message it came from, and every confidence number is computed by formula from how many falsification probes it survived. Self-claims like “i’m an INTJ” get their own reconciliation stream, checked against behaviour.

Fully local: no telemetry, no remote calls, no cloud sync; the HTTP server only listens on 127.0.0.1.

Built by191-iota
04
PYTHON · DOCS & AUDIO
PythonAnthropic APIfaster-whisper

synthesis /synthesis ↗

Point it at a folder of documents and get a self-contained interactive HTML briefing, or live-transcribe a lecture or meeting into a structured report.

In document mode the pipeline ingests files (PDF via pdftotext, DOCX via pandoc, GitHub repos via a github: prefix), runs a two-pass extraction plus an independent confirmation pass, and renders one HTML file: a vertical timeline with a TODAY line, expandable cards, and a game plan. Only items both passes agree on and that carry a grounded source quote make it through. Live mode transcribes microphone or system audio with faster-whisper.

An item ships only if both passes agree on it and it carries a grounded source quote; live audio is processed in memory and never written to disk.

Built by191-iota
05
AGENT SCAFFOLD · ANY DOMAIN
Claude CodeAgentMarkdown

The Factory /the-factory ↗

Drop source material into context/, write task.md, and run Claude Code as an autonomous worker that does the job end to end.

The agent reads your context folder and task, plans, writes output to a new folder under deliverable/, and reviews it end to end. Before and during the work it keeps its own notes in notes/: what is where, how it connects, what state it is in. The next run reads the notes first and resumes warm. Run it interactively or headless with --auto.

Output only ever lands in a new deliverable/ folder; your context/ is left untouched.

Built by191-iota

RUST + VUE · REAL-TIME
RustVue 3WebSocketPostgreSQL

Neuro /neuro ↗

Run five short, gamified cognitive tests (reaction time, working memory, processing speed, and more) and get a five-part score profile with per-game leaderboards.

Neuro administers tasks modelled on established psychometric paradigms over a real-time WebSocket pipeline (Rust/Actix-Web backend, Vue 3 frontend) and records fine-grained metrics to PostgreSQL. Each game targets one narrow ability from the Cattell-Horn-Carroll model, so you get five separate ability scores. Per-game leaderboards let you compare against other users.

Five separate ability scores, with anticipatory and lapse trials dropped so each one stays meaningful.

07
RUST · DISCORD BOT
RustDiscordSQLiteplotters

ClockBot /clock ↗

Clock in and out of work activities from Discord slash commands and get weekly hours, rankings, and charts back.

ClockBot tracks time per activity in a Discord server. /clock in starts a session, /clock out stops it and reports the duration, and who, leaderboard, and stats read back current sessions, rankings, and breakdowns. /clock chart renders a PNG line chart of the top users’ weekly hours over 1 to 52 weeks. Sessions live in a SQLite file, and weekly stats archive every Monday at 00:00 Europe/Zurich.

Charts render with plotters on the pure-Rust ab_glyph backend and an embedded font, so the runtime container needs no system font packages.

RUST · CLI
RustCLIi18nJSON

transl /transl_cli ↗

Add a key and value to your base-language JSON file and have transl carry that key across every other language file, so your translation files stay in sync.

A small Rust CLI for projects that keep one JSON file per language. Run transl <header> <key> <value> <base_lang> to add a property to the base language, and it syncs the new key into the others. Installs with cargo install --path .; prebuilt Windows binaries are on Releases.

A single MIT-licensed binary that touches only your language JSON files.

Built by191-iota
09
NEXT.JS · LOCAL-ONLY
Next.js 16React 19Tailwind 4

money-counter /money-counter ↗

Track savings goals by name, target, current amount, and monthly contribution, and see how many months each one needs to hit its target.

Each counter holds a name, a target, a current balance, and a monthly contribution. It shows percent progress and the amount left, and computes an ETA by dividing what remains by the monthly contribution and rounding up. Quick buttons add 100, 500, or 1000 or subtract 100; a summary row totals saved against target across all goals.

No backend and no accounts: counters live in your browser’s localStorage and never leave the device.

The bench rules

How they’re built

Not every tool does every one of these. Where a tool makes a claim or touches your data, these hold.

01
Receipts

A claim about a person, a relationship, or the law carries a quote and a pointer back to the source, or it does not get written down.

overseer · saul-swissman · auspex

02
Computed confidence

Confidence is computed from counted evidence and shown with its tally. auspex derives it by formula from falsification probes; Overseer shows the dated tally for you to judge.

auspex · overseer

03
Explicit about what leaves

Your files stay on disk. auspex sends nothing off your machine; the tools that do reach a model strip secrets first and say what goes out.

auspex · synthesis · overseer

04
No ceremony

Several tools are one Markdown protocol with the model as the engine. Nothing to deploy, no schema, no service to babysit.

overseer · saul-swissman
